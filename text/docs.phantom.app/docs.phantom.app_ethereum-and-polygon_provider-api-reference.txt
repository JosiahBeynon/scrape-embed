This same provider is made available at window.phantom.ethereum to prevent namespace collisions. This API is how a dApp will make requests to the user; reading account data, connecting to the website, signing messages, and sending transactions will all be done through this provider object.

This provider API is specified in greater detail in EIP-1193.
